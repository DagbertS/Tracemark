Tracemark Developer Quickstart

Add AI governance to your app in under 5 minutes

v0.1.0-mvp

How It Works

Tracemark is a drop-in reverse proxy that sits between your application and your LLM provider. Change your base_url to point at Tracemark, add two identification headers, and every AI call is automatically governed.

1

Point your SDK at Tracemark

Change the base_url in your existing OpenAI/Anthropic client to your Tracemark endpoint.

2

Add caller identification

Set X-Tracemark-Caller and X-Tracemark-Tenant headers to identify your app and organization.

3

Send requests as normal

Tracemark intercepts, applies policies (PII masking, topic blocklist, confidence checks), logs to the provenance trail, and forwards to the model.

Node.js

from openai import OpenAI

client = OpenAI(
    api_key="your-openai-api-key",
    base_url="https://tracemark-production.up.railway.app/v1",
    default_headers={
        "X-Tracemark-Caller": "my-app-name",
        "X-Tracemark-Tenant": "your-tenant-slug",
    }
)

response = client.chat.completions.create(
    model="openai/gpt-4o",  # prefix with provider
    messages=[{"role": "user", "content": "Hello"}]
)

print(response.choices[0].message.content)
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'your-openai-api-key',
  baseURL: 'https://tracemark-production.up.railway.app/v1',
  defaultHeaders: {
    'X-Tracemark-Caller': 'my-app-name',
    'X-Tracemark-Tenant': 'your-tenant-slug',
  },
});

const response = await client.chat.completions.create({
  model: 'openai/gpt-4o',
  messages: [{ role: 'user', content: 'Hello' }],
});

console.log(response.choices[0].message.content);
curl -X POST https://tracemark-production.up.railway.app/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-openai-api-key" \
  -H "X-Tracemark-Caller: my-app-name" \
  -H "X-Tracemark-Tenant: your-tenant-slug" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [{"role": "user", "content": "Hello"}]
  }'

Anthropic Native Format

For applications using the Anthropic SDK, Tracemark also exposes the native /v1/messages endpoint:

import anthropic

client = anthropic.Anthropic(
    api_key="your-anthropic-key",
    base_url="https://tracemark-production.up.railway.app",
)

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}],
    extra_headers={
        "X-Tracemark-Caller": "my-app",
        "X-Tracemark-Tenant": "your-tenant",
    }
)

Multi-Provider Routing

Prefix your model name with the provider to route through Tracemark to any supported LLM:

OpenAI

openai/gpt-4o
openai/gpt-4o-mini

Default if no prefix.

Anthropic

anthropic/claude-sonnet-4-6
anthropic/claude-haiku

Also via /v1/messages

Google

google/gemini-pro
google/gemini-1.5-flash

Via Gemini API

Headers Reference

HeaderRequiredDescription
X-Tracemark-Caller Recommended Identifies your application (e.g., "billing-service", "chatbot-v2")
X-Tracemark-Tenant Recommended Identifies the organization/company slug (e.g., "acme-corp")
X-Tracemark-Session Optional Groups related requests into a session for analysis
Authorization Yes Your LLM provider API key (forwarded to upstream)

What to Expect

After integrating, every AI call through Tracemark will:

1. Scan for policy violations — PII (emails, phone numbers, credit cards, IBAN), forbidden topics, and confidence issues are detected automatically.

2. Sanitize sensitive data — PII is replaced with stable tokens (e.g., [EMAIL_a3f2]) before reaching the model, then restored in the response.

3. Log to provenance trail — Every request gets an HMAC-SHA256 chained audit entry with full metadata (model, tokens, cost, policies applied).

4. Fire connectors — Violations trigger Slack alerts, email notifications, or webhook calls to your enterprise systems.

Response Headers

Tracemark adds these headers to every response:

Tracemark-Entry-ID Tracemark-Verdict Tracemark-Entities-Masked Tracemark-Remediation-Triggered

Try It Now

Send a test request with PII to see Tracemark in action:

curl -s -X POST https://tracemark-production.up.railway.app/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-Tracemark-Caller: quickstart-test" \
  -H "X-Tracemark-Tenant: demo" \
  -d '{
    "model": "gpt-4o",
    "messages": [{
      "role": "user",
      "content": "Please help John Doe at j.doe@email.com with his refund"
    }]
  }' | python3 -m json.tool

Expected result: The request will be blocked (PII detected: email, name). Check the Live Feed in the dashboard to see the entry with verdict=BLOCKED and the matched PII patterns.
